Competition Creator

System requirements

The competition creator runs on Windows (Vista, Windows 7, Windows 8). It requires the .NET framework 4 to be installed. When this is not installed, it can be installed for free from here. No special HW requirements are imposed. If your computer is fast, the optimization process will go faster, and vice versa. Having a large screen is a pro, since the Competition Creator can show a lot of information at the same time on the screen.

Installation

There are different ways to install the Competition Creator:

  • Zip-file: Simply extract the file somewhere on your system. The CompetitionCreator file can immediately be started.
  • From competitioncreator.doren.be: This is slightly more complex, for the following reasons:
    • The virus-scanner has to be turned of during download. It does not trust executables coming from the web, and it blocks the execution of it. If your virus-scanner is not temporarily disabled, the installation process ends with a 'Cannot start application' because the download did not succeed.
    • Windows does not trust the executable. It is not 'signed', so that windows can check its origin. How windows handles this is dependent on the version:
      • Windows Vista: ??
      • Windows 7: When pressing install, an executable is downloaded. Depending on your browser you will be asked whether you really want to execute the program because it can potentially 'Harm' your computere.
      • Windows 8: ??

License

Competition creator has a license that is restricted to a single computer. When the application is installed on a computer, an license can be requested. Based on a unique hardware Id a license is created with an expiring date. Typically, the license is valid for one year.

Without license, the tool is fully functional except that the data cannot be saved or exported. So you can expiriment with the capabilities: import registrations, optimize a competition, but not saving or exporting the generated competition.

Updates

The Competition Creator checks for updates that are published when starting the application. When you decide to update, turn your virus-scanner temporarily off, and update. When updates are published these are backwards compatible, i.e. your existing data can still be used.

Registrations input

The registrations input is an XML based file that contains all basic information on team subscriptions:

  • Clubs, and per club:
    • Id: Integer used as reference for the club. Typically, the key-index used in the database that handles the subscriptions
    • Name: Name of the club
    • StamNumber: This is a unique string that represents the club. It becomes essential when importing other competitions in which the club also participate. In this import, this stamNumber should be identical otherwise the tool cannot recognize it is the same team.
    • Teams that are subscribed, per team:
      • Id
      • Name
      • Group (X or Y)
      • SerieId: Serie in which this team is registrated
      • SerieName: Serie in which this team is registrated
      • Day (6=saterday, 7=sunday): Default day on which the team plays
      • StartTime: Default time on which the team plays
      • SporthallId: Sporthall in which the team plays. It must be one of the sporthalls that is used by the club
      • ContactEmail: Who is the contact person.
    • Sporthalls that are used by the club. Per sporthal:
      • Id
      • Name
      • The days that the sporthall is not available.
    • Free format text: Allowing clubs to enter free format text that can be shown in the Competition Creator. Typically not all constraints can be handled programatically, and this allows for a backdoor.

Distances input

When more than one poule is created in a serie, there is freedom how to divide the teams in this serie over de poules. One of the options is to divide it optimally based on the distance, i.e. cluster the teams in a poule that live close together. For this function the distances between sporthalls are required. Typically, this is information that can be retrieved from Google Maps, but due to limitations on the number of requests that is allowed per day, this information has to be stored and feed into the program. The distances input contains:

  • Sporthalls, and per sporthal:
    • Id
    • Name
    • Adress
    • City
    • Latitude: Coordinate to enable displaying the sporthal on google maps
    • Longitude: Coordinate to enable displaying the sporthal on google maps
  • Distances, and per distance
    • Id1: Id of first sporthal
    • Id2: Id of second sporthal
    • Time: Time it takes to travel from the first sporthal to the second in minutes
    • Distance: Distance between the two sporthalls.

The list of distances is bi-directional. So both the distance from 1 to 2, and the distance from 2 to 1 must be included in the list. When the list of distances is not complete for all sporthalls, or is completely absent in the distances input, the distances between sporthalls will be calculated on the Latitude and Longitude. This will be less accurate, but sufficient to have a good division in poules. Note that when the distances are not specified, the section "<Distances></Distances> must be present anyhow.

An example of a distances input XML can be found here.

Ranking input

When in the second half of the year series of the competition are re-divided based on the ranking of the first half, the ranking can be imported. After the ranking is imported, it can be used to sort the poules within a serie to easily re-create the poules in a serie based on the rankings. The ranking is again a XML file with the following information:

  • Rankings, with for each ranking:
    • SerieId: Id of the Serie the ranking belongs to
    • SerieName: Name of the Serie the ranking belongs to
    • PouleName: Name of the Poule the ranking belongs to
    • TeamId: The Id of the Team where this ranking is from
    • TeamName: The Name of the Team
    • Ranking: The actual ranking (1, 2, ....)
    • Score: The number of points that a team has achieved. This number is used to determine for teams from different poules that have the same ranking who is the best.
